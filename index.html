Kotlin In Android

Kotlin是什么

一种基于 JVM 的静态类型编程语言,旨在提供一种更好的 Java 开发方式，同时支持Android、后端、IOS、Web开发.

为什么要使用 Kotlin

  • Kotlin 上手比较容易，基本过一遍官方文档就可以上手开发
  • Kotlin 能明显提高Android开发者的开发效率；
  • 作为 Android 官方语言，Goodle对于Kotlin提供了更好的支持，并且Google P 以及 Gradle 新版本的源码都将加入 Kotlin 的元素。

Kotlin vs Java

  • 编译过程 对比

Java 是一种基于 JVM 的语言，Java 语言通过 词法分析 - 语法分析 - 语义分析 等一系列步骤生成JVM可识别的Java字节码，最终由JVM解释执行

Kotlin 也是一种基于 JVM 的语言，它的出现是为了提供一种更好更优雅的 Java 开发方式，通过简单分析 Kotlinc 的编译脚本，发现在编译生成中间代码的过程中两者基本一致，而区别是存在于生成目标代码也就是 Java字节码的过程中，Kotlin 在这个环节做了很多类似于封装的事情，比如 Getter/Setter 代码的生成Companion转变成静态类以及其它一些Kotlin特性解析的过程,可以说大部分的Kotlin让Java开发起来更加优雅的特性都是在这个环节转换完成的。

Kotlin 实践

通过这段时间的开发学习，对Kotlin对于开发效率的提升感受十分明显。现将简单介绍下Kotlin 在Android开发中常用的特性以及使用Kotlin期间碰到的问题。

  • 优雅的特性

    • 告别NPE以及自动识别类型

    • 在 Kotlin 中定义的变量默认都非空（如果想定义可空对象，需要在对象类型后面加上 ？），这样 Kotlin 就会在编译阶段检查所有的可能出现的空指针的风险。如下：

      var postsAdapter: PostAdapter?
    • Kotlin 声明赋值在同一条语句的时候可以自动推断类型，无需显示声明。用val 定义final，用var 定义普通变量。如下：
       var count: Int = 0    // Int 可以省略
var str: String = "hello"   //String 可以省略
   val flag = true  //final 变量

  • 数据类、对象类、自动生成getter/setter

    • Java bean对象可以直接使用 data class来完成。如下：
    data class Post(
val id: Int,
val userId: Int,
val title: String,
val body: String)
    • Java 常用的单例模式或者static 类/方法都可以 object class来完成。如下：
    object BeanClass {
      fun test1() {
      }
      @JvmStatic
      fun test2() {
      }
  }

  • 需要注意的问题

    • Kotlin 不再有 Checked Exceptions。

    虽然Java的Checked Exceptions 写起来很繁琐，但是有助于提高代码的健壮性，Kotlin抛弃了Checked Exceptions，这就对于开发者处理异常有了更高的要求。

    • 和开源框架的兼容性问题
    • 小桔养车App中使用到了Arouter，在使用annotationProcessor的情况下 Kotlin代码在编译的时候不能生成注解文件，不过kapt(Annotation Processing for Kotlin) 很好的解决了这个问题，Arouter也做了适配。
    • 小桔养车App中使用到了fastJson,使得用Kotlin data class写的Bean类转换过程中会报错，看fastjson的issues介绍fastjson最新的版本做了兼容，不过经过测试还是会有问题，所以App里面的数据类暂时还都是用Java写的。
    • 不过随着使用Kotlin开发Android的人越来越多，适配问题应该都会逐渐解决。
    • 性能以及代码质量
    • 由于运行时和Java都是基于JVM的字节码解释执行，所以运行时性能不会有什么影响。编译时理论上由于会生成很多封装代码会导致编译变慢，但是在开发过程中感受不深，不过看其它开发者做了一些专门测试的经验文章说在增量编译的时候和Java基本没什么差别，在某些task上甚至会比Java更快。
    • 代码质量可以使用Klint、detekt等工具去做些检查，也有助于更好的理解Kotlin。

总结

经过一个多月的Kotlin开发,体验到了Kotlin的优雅也碰到了Kotlin的坑，希望Kotlin的发展越来越好，也期待更多的伙伴加入到Kotlin中开发中来。
